Programmation structurée et non structurée

Introduction

La programmation non structurée est historiquement le paradigme de programmation le plus ancien et capable de créer des algorithmes Turing-complet (voir aussi machine de Turing).

Un programme écrit de façon non structurée utilise des instructions de saut (c.-à-d. l’instruction goto) vers des étiquettes ou vers des adresses d’instructions. Les lignes de ce programme sont généralement numérotées (comme dans le langage BASIC) ou peuvent avoir des étiquettes : cela permet au flux d’exécution de sauter (c.-à-d. brancher) à n’importe quelle ligne du programme. Ceci est en contraste avec la programmation structurée qui utilise des constructions de bloc de sélection (if/then/else) et de répétition (while et for).

Il existe des langages de programmation de haut et de bas niveau qui utilisent une programmation non structurée. Parmi les langages communément classés comme non-structurés, on peut citer le JOSS, le FOCAL, le TELCOMP, les langages assembleurs, les fichiers de commande MS-DOS, ainsi que les premières versions du Fortran, du BASIC, du COBOL et de MUMPS.

Bien que cela vous ait été caché, vous avez déjà fait de la programmation structurée en algorithmique ! En effet, une instruction if réalise un saut sans que vous vous en apercevez. Par exemple, examinons le code C suivant :

if (/* Condition */)
{
   /* Bloc */
}

/* Suite du programme */

Dans le cas où la condition est fausse, l’exécution du programme passe le bloc (délimité par des accolades) de l’instruction if et exécute la suite du programme. Autrement dit, il y a un saut / branchement vers le bloc suivant. En programmation non-structurée le programmeur (vous) introduit explicitement des instructions de saut dans le programme. Ainsi, l’exemple précédent dans un format non-structuré devient :

if ( /* SAUF Condition */) goto FI;

/* Bloc */

FI:

/* Suite du programme */

Ici, une instruction goto avec l’étiquette associée (c.-à-d. l’étiquette FI) sont introduites dans le programme pour accomplir la même tâche que précédemment. Notons, toutefois, que la condition de l’instruction if est inversée ici (le mot clé SAUF) pour réaliser le même comportement que dans le cas de la programmation structurée (et la suppression des accolades qui ne sont plus utiles dans ce cas).

Une instruction goto en programmation C permet de sauter inconditionnellement du goto à une instruction étiquetée dans la même fonction. La syntaxe pour une instruction goto en C est la suivante -

goto label;
..
.
label: instruction;

Ici, label peut être n’importe quel chaîne alphanumérique (à l’exception des mots-clé C), et elle peut être définie n’importe où dans une fonction C au-dessus ou au-dessous d’une instruction goto.

Exemples de codes structurés et non-structurés

if / then / else

Code structuré Code non structuré 
if( /* Condition */ )
{
 /* Bloc 1 */ 
}
else
{
 /* Bloc 2 */ 
}

 /* Suite du programme */ 

if( /* SAUF Condition */ ) goto ESLE;

 /* Bloc 1 */
goto FI;
ESLE:

 /* Bloc 2 */

FI:
 /* Suite du programme */

Ici, deux instructions goto sont introduites : le premier goto (c.-à-d. goto ESLE) permet de brancher sur le bloc du else si la condition est fausse. Dans le cas où la condition est vraie, le bloc then (c.-à-d. le Bloc 1) est exécuté et le deuxième goto à la fin de ce bloc (c.-à-d. goto FI) permet de sauter à la suite du programme et ne pas exécuter le bloc du else (c.-à-d. le Bloc 2).

switch / case

Une approche naïve pour convertir un bloc de switch / case en un programme non structuré serait de réécrire le bloc de code en une série de if / then / else et procéder par la suite à la conversion de ces blocs comme décrit plus haut. Une autre approche, plus efficace, utiliserait les étapes suivantes :

  • Convertir chaque case i en une étiquette unique (ex: case 5 deviendra ESAC5).

  • Créer un tableau avant l’instruction switch pour contenir toutes les valeurs des étiquettes correspondant aux instructions de case / default dans le bloc du switch (c.-à-d. les étiquettes ESAC0, ESAC1 …).

  • Créer une étiquette à la fin du bloc du switch / case et convertir chaque instruction de break du switch en une instruction de goto vers cette nouvelle étiquette pour sauter vers la fin du bloc (c.-à-d. l’instruction goto KAERB dans le code ci-dessus).

  • Enfin, en fonction de la valeur de l’expression du switch, nous récupérerons l’adresse de l’étiquette à partir du tableau et nous effectuons le saut adéquat à l’aide d’une instruction goto.

Code structuré Code non structuré 

switch( /* Expression */ )
{
  case 0: /* Bloc 0 */
          break;
  case 1: /* Bloc 1 */
          break;
   ... 
  case N: /* Bloc N */
          break;
  default: /* Default */          
}

/* Suite du programme */

void *SwitchTab[] = { &&ESAC0, &&ESAC1, ..., &&ESACN, &&TLUAFED };
goto *SwitchTab[ /* Expression */ ];

ESAC0: /* Bloc 0 */
goto KAERB;
ESAC1: /* Bloc 1 */
goto KAERB;
...
ESACN: /* Bloc N */
goto KAERB;
TLUAFED: /* Default */         
KAERB:

/* Suite du programme */

REMARQUE : le schéma décrit ci-dessus suppose l’utilisation du compilateur GCC qui offre une fonctionnalité appelée “labels as values” - D’autres compilateurs C pourraient utilisés d’autres méthodes pour la conversion.

Boucles ‘while’ et ‘for’

La conversion d’une boucle while en un code non structuré est très similaire à la conversion d’un bloc if / then. En effet, une petite comparaison entre le code ci-dessus et celui du bloc if / then introduit plus haut montre qu’on plus de l’inversion de la condition (c.-à-d. SAUF Condition), nous procédons ici à la substitution du mot clé while par le mot clé if et nous insérons un deuxième goto à la fin du bloc de la boucle, juste avant l’étiquette de sortie pour retourner à nouveau au test de la condition associée à la boucle.

Code structuré Code non structuré 

while( /* Condition */ )
{
 /* Bloc */ 
}

 /* Suite du programme */ 

ELIHW:
if( /* SAUF Condition */ ) goto SUITE;

 /* Bloc */
goto ELIHW;
SUITE:
 /* Suite du programme */

Pour convertir une boucle for en un code non structuré, on notera que cette boucle est conceptuellement équivalente à une boucle while. En effet, il est possible/facile de réécrire toute boucle for dans le langage C en une boucle while (voir l’exemple en dessous).

boucle for boucle while 

for( i = 0; i < N; i++  )
{
    /* Bloc */

}

/* Suite du programme */
⇔ 
i = 0;
while( i < N )
{
    /* Bloc */
    i++;
}

/* Suite du programme */

Boucle do / while

Enfin, comme pour la boucle while vue plus haut, la transformation vers un code non structuré de la boucle do / while requiert le remplacement du mot clé while par le mot clé if. Notons toutefois, que la condition de la boucle reste inchangée (c.-à-d. non inversée). Notons aussi qu’un seul goto est introduit pour retourner au début de la boucle do / while.

Code structuré Code non structuré 
do
{
    /* Bloc */
}
while( /* Condition */ );

/* Suite du programme */

OD:

/* Bloc */

if( /* Condition */ ) goto OD;

/* Suite du programme */
