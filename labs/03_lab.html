Programmation « non structurée »

Publié le Dimanche 15 Octobre 2023

Échéance : Dimanche 22 Octobre 2023

Télécharger [démarrage]


Objectifs

  • Rappel sur les structures de contrôle dans le langage C

  • Connaître la programmation « non structurée »

Préambule

Toute personne qui s’initie à la plongé sous-marine commence d’abord par apprendre les bases de cette technique dans une piscine ! Similairement, avant de s’attaquer à la programmation en assembleur (quelque chose qui effraie beaucoup d’étudiants), les exercices suivants vous apprennent à écrire des programmes simples dans le langage C mais qui se rapproche un pas de l’assembleur.

Commencez d’abord par télécharger le fichier de démarrage (voir plus haut dans ce document) et décompressez son contenu dans le répertoire de votre choix. Dans ce TP, il est demandé d’écrire des solutions dans le langage C aux problèmes donnés ci-dessous avec la contrainte suivante : Aucun mot clé for, while, do, switch ou else ne peut être utilisé dans vos programmes.

Pour vérifier l’exactitude de vos implémentations dans les exercices ci-dessous, configurez puis compilez les codes fournis à l’aide des commandes suivantes (sous environnement UN*X) :

$ cd chemin/vers/le/repertoire/lab3
$ mkdir build      # étape de configuration
$ cd build         # à réaliser la première 
$ cmake ..         # fois seulement
$ make             # <- compilation - exécutez cette commande à chaque fois que vous modifiez vos fichiers ".c"

Si la compilation se passe sans erreurs, vous pouvez exécuter les tests de validation à l’aide de la commande

$ ctest --output-on-failure

Cette commande affichera sur votre écran le résultat de quelques tests qui vont vous permettre de vérifier si vous avez implémenté correctement les fonctions sus-citées.

Indication : Écrivez d’abord vos algorithmes d’une manière standard (c.-à-d. avec les mots-clés for, while, …), ensuite vous pouvez les convertir en des codes sans ces mots clés en vous aidant des exemples et indications donnés dans le document “Programmation structurée et non structurée”.

Exercice 1 : (Somme des carrés)

En mathématiques, la somme des carrés de N nombres entiers est décrit comme suit :

\[sum = \sum_{i=0}^{N-1} n_i^2\]

où \(n_0, n_1, ..., n_{N-1}\) sont des nombres entiers.

Ecrivez le corp de la fonction SumOfSquares(int count, const int *tab) dans le fichier src/SumOfSquares.c qui retourne la somme des carrés des éléments d’un tableau d’entiers. Le paramètre count est le nombre des éléments du tableau, et tab est un pointeur sur un tableau de type int.

Contraintes à respecter :

  • mot-clés interdits : for, while, do, switch, else
  • # vvv test de vérification des contraintes  vvv   
lab3/src$ python3 test.py --filename=SumOfSquares.c --for --while --switch --else

Exercice 2 : (StrCmp)

Dans la bibliothèque standard du langage C, la fonction strcmp() (cf. man 3 strcmp) compare, caractère par caractère, deux chaînes en mémoire pour établir quelle chaîne de caractères vient en premier dans l’ordre lexicographique standard, c.-à-d. en fonction des valeurs ASCII des caractères. Voici quelques exemples :

  • “a” < “b”
  • “abc” < “abcd”
  • “A” < “a”

Les chaînes de caractères à comparer sont représentées par des octets contigus en mémoire (chaque octet est un caractère ASCII) suivi du caractère NUL (0x00).

Ecrivez le corp de la fonction StrCmp(const char *s1, const char *s2) dans le fichier src/StrCmp.c qui compare deux chaînes de caractères. Les arguments s1 et s2 de la fonction StrCmp(...) sont les adresses des deux chaînes à comparer. L’adresse mémoire du début de la première chaîne est dans s1, et l’adresse mémoire du début de la deuxième chaîne est dans s2. Si la première chaîne est supérieure à la seconde, renvoyez un nombre positif. Si la deuxième chaîne est plus grande, renvoyez un nombre négatif. Si les chaînes sont égales, renvoyez la valeur 0.

Contraintes à respecter :

  • mot-clés interdits : for, while, do, switch, else
  • # vvv test de vérification des contraintes  vvv   
lab3/src$ python3 test.py --filename=StrCmp.c --for --while --switch --else

Exercice 3 : (Calculatrice)

Ecrivez le corp de la fonction calculatrice(float nbr1, float nbr2, char op) dans le fichier src/calc.c qui retourne le résultat de l’opération nbr1 op nbr2, où nbr1 et nbr2 sont deux nombres de type float, et op est un caractère indicant le type de l’opération :

  '+' : opération d'addition
  '-' : opération de soustraction
  '*' : opération de multiplication
  '/' : opération de division

si op est différent des quatre opérations indiquées ci-dessus alors la fonction doit retourner la valeur NAN (cf. man 3 nan).

Contraintes à respecter :

  • mot-clés interdits : for, while, do, switch, else
  • # vvv test de vérification des contraintes  vvv   
lab3/src$ python3 test.py --filename=calc.c --for --while --switch --else
